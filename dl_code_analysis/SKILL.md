# 深度学习代码仓库技术理解文档生成器

## 概述

本 Skill 用于对深度学习/计算机视觉代码仓库进行深度分析，生成面向研究者和开发者的**中文 Markdown 技术理解文档**。文档聚焦：技术原理、代码实现细节、张量维度流分析，帮助读者真正理解代码在做什么、为什么这样设计、具体是怎么实现的。

## 触发条件

当用户要求分析代码仓库、生成技术文档、理解模型架构、梳理数据流或张量维度时触发。触发词包括：代码分析、技术文档、架构分析、张量流、理解代码、dl-analysis 等。

## 执行流程

### 第一步：代码仓库探索

1. **确定项目入口**：找到训练脚本（如 `train*.py`）、推理脚本（如 `eval*.py`、`decode*.py`）、主模块文件
2. **定位核心模块**：搜索包含 `Model`、`Network`、`Encoder`、`Decoder`、`Attention`、`ContextModel`、`EntropyModel`、`Transformer`、`Block` 等关键类的文件
3. **理解项目结构**：通过目录结构和配置文件了解整体组织
4. **阅读已有文档**：如有 README、论文说明等，结合理解但以代码为准

### 第二步：深度代码阅读

优先阅读以下文件（按重要性排序）：
1. **模型定义文件**：包含核心网络结构的文件
2. **核心模块实现**：重点关注 `forward`、`inference`、`compress`、`decompress`、`encode`、`decode`、`update`、`insert`、`build`、缓存初始化等函数
3. **训练与推理入口**：分析训练/推理流程差异
4. **配置文件**：超参数来源（通道数、层数、窗口大小、head 数、slice 数等）
5. **工具函数**：张量变形、窗口划分、mask、概率分布、熵编码、缓存管理等

### 第三步：生成技术理解文档

按照 [reference.md](reference.md) 中的文档结构生成完整的中文 Markdown 技术理解文档。

## 核心写作原则

1. **公式格式**：行内公式用 `$公式$`，独立公式用 `$$公式$$`，禁止使用 `\(...\)` 或 `\[...\]`
2. **维度必写**：每个变量必须写明形状和每个维度的语义
3. **不跳步**：缓存、reshape、permute、mask、索引等操作必须展开说明
4. **训练/推理区分**：明确指出二者差异
5. **准确性优先**：基于代码分析，不确定的标注"代码中未能完全确认"，发现不一致以代码为准
6. **语义具体化**：禁止"这里进行 attention 计算"等笼统写法，必须说明 Q/K/V 来源、矩阵形状、元素含义

## 输出要求

- 语言：中文
- 格式：Markdown
- 公式：LaTeX（`$...$` 和 `$$...$$`）
- 文档结构：遵循 reference.md 中的 13 节模板
- 准确性：严格基于代码，不编造
